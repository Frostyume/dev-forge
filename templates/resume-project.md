# 简历项目模板（STAR，中英双骨架）

按 `lang` 取对应骨架，把占位填实。source_problems 记录本稿引用了哪些已解决问题，便于增量更新。遵守 conventions §7/§8。

## 中文骨架（lang: zh）

```
---
type: resume-project
lang: zh
updated: <YYYY-MM-DD>
source_problems: [<P-id, ...>]
---
# <项目名> — <一句话定位（做什么、为谁创造什么价值）>
**技术栈**：<语言 / 框架 / 关键库>
**角色 / 周期**：<独立开发 / 核心贡献者 · YYYY.MM–YYYY.MM>
## 项目简介
<2–3 句：项目目标、你负责的部分、整体成果>
## 关键贡献（STAR）
- **<主题，如"修复梯度爆炸导致的训练崩溃">**：<背景：当 …时（S/T）>，定位到 <根因>（A），通过 <具体做法/方法>，使 <可量化结果，如训练稳定性从 X 提升到 Y / 排障时间从 N 降到 M>（R）。
- **<主题2>**：…
## 亮点速记（可直接粘进简历的单行 bullet）
- <动词开头，含量化结果，一行一条>
- …
```

## English skeleton (lang: en)

```
---
type: resume-project
lang: en
updated: <YYYY-MM-DD>
source_problems: [<P-id, ...>]
---
# <Project Name> — <one-line positioning: what it does, value delivered>
**Stack**: <languages / frameworks / key libs>
**Role / Duration**: <Solo dev / Core contributor · YYYY.MM–YYYY.MM>
## Overview
<2–3 sentences: goal, your scope, outcome>
## Key Contributions (STAR)
- **<theme, e.g. "Eliminated training crashes from exploding gradients">**: When <situation/task>, root-caused <cause> and <action: concrete method/tooling>, which <quantified result, e.g. cut debug time from N to M / raised stability from X to Y>.
- **<theme 2>**: …
## Resume-ready bullets (paste directly)
- <Action verb + quantified result, one per line>
- …
```
